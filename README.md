# STM32 串口转 RS485 通信实验

本项目是一个基于 STM32F103C8T6 的 RS485 通信实验项目。通过 MAX3485 芯片将单片机的 TTL 串口信号转换为 RS485 差分信号，并与电脑端的 RS485 转 USB 模块进行双工通信。

## 硬件清单
1.  **STM32F103C8T6** 开发板 (最小系统板)
2.  **MAX3485 模块** (TTL 转 RS485，支持 3.3V 供电)
3.  **RS485 转 USB 模块** (蓝色外壳，常见的工业/实验室转换器)
4.  杜邦线若干

## 硬件接线图

### 1. STM32 与 MAX3485 封装模块连接
根据您的模块封装（DI/RO 换成了 A/B，DE&RE 换成了 EN），接线定义如下：

| STM32 引脚 | MAX3485 模块引脚 | 说明 |
| :--- | :--- | :--- |
| **PB6 (TX)** | **A (即 DI)** | 串口发送数据输入 |
| **PB7 (RX)** | **B (即 RO)** | 串口接收数据输出 |
| **PB12** | **EN (即RE#和DE)** | **方向控制** (高电平发送，低电平接收) |
| **3.3V** | **VCC** | **必须接 3.3V** |
| **GND** | **GND** | 共地 |

> **注意**：
> *   **PB6 和 PB7** 使用了引脚重映射功能 (Remap)，具体配置见 `usart.c`。
> *   该模块将标准的 DI/RO 丝印为了 A/B，请务必区分这里的 A/B（TTL侧）与 RS485 总线侧的 A/B。

### 2. MAX3485 与 RS485 转 USB 模块连接
虽然 RS485 是差分信号，但为了提高抗干扰能力和保证共模电压范围，建议连接 GND。

| MAX3485 模块总线侧 | RS485 转 USB 模块 | 说明 |
| :--- | :--- | :--- |
| **A+** | **A+ / A** | 差分信号正极 |
| **B-** | **B- / B** | 差分信号负极 |
| **GND** | **GND** | **共地线** (增强远距离传输稳定性) |

## 软件实现逻辑

### 1. 串口配置
*   **波特率**：9600
*   **数据位**：8位
*   **停止位**：1位
*   **校验位**：无
*   **中断**：开启空闲中断接收 (`HAL_UARTEx_ReceiveToIdle_IT`)

### 2. RS485 半双工切换
由于 RS485 是半双工通信，必须手动控制 PB12 引脚来切换收发模式：
*   **发送前**：`HAL_GPIO_WritePin(GPIOB, GPIO_PIN_12, GPIO_PIN_SET);` (进入发送模式)
*   **发送后**：`HAL_GPIO_WritePin(GPIOB, GPIO_PIN_12, GPIO_PIN_RESET);` (恢复接收模式)

### 3. 复位乱码优化
为了消除单片机复位瞬时电平抖动产生的乱码 (如 `?Hello`)，代码中加入了以下逻辑：
*   **上电延迟**：`HAL_Delay(100);` 等待电平稳定。
*   **清除标志**：`__HAL_UART_CLEAR_FLAG(&huart1, UART_FLAG_TC);` 清除上电产生的虚假发送完成标志。

## 通信测试
1.  将 RS485 转 USB 模块插入电脑。
2.  打开串口调试助手，选择对应的 **COM口** (RS485转USB的端口)。
3.  设置波特率为 **9600**。
4.  复位单片机，串口助手将收到 `Hello, World!`。
5.  在串口助手发送任意字符，单片机将通过中断接收并原样回传（Echo）。

## 核心技术问答 (FAQ)

**Q: 为什么工业上普遍使用 RS485？是因为稳定性吗？**
A: 是的，稳定性是核心原因。RS485 的稳定性源于以下技术优势：
1.  **抗干扰能力强 (差分信号)**：RS485 使用 A、B 两根线的电压差传输信号。外界电磁干扰通常会同时影响 A/B 线（共模干扰），但由于差值保持不变，信号依然能被准确识别。
2.  **传输距离远**：在低波特率下，理论传输距离可达 **1200 米**，远超 TTL (1m) 或 RS232 (15m)。
3.  **支持多节点组网**：一条总线上可以挂载多个设备（通常为 32 个节点），适合工业传感器和仪表的分布式部署。
4.  **成本低廉**：仅需两根线（双绞线）即可实现长距离通信，布线简单且硬件芯片方案非常成熟。

**Q: 为什么必须用 RS485 转 USB 模块？**
A: 单片机输出的是 TTL 电平（对地电压），而 RS485 使用的是差分电压信号。两者物理特性不同，必须经过专门的转换芯片和模块才能让电脑识别。

**Q: 为什么复位时会出现乱码？**
A: 主要由于上电瞬间 IO 口电平抖动或初始化时的瞬态信号被串口助手误识别。本项目通过 **100ms 延迟** 和 **清除硬件发送标志 (TC)** 的方法解决了此问题。

**Q: 为什么发送字符长度要减 1？**
A: 在 C 语言中，`"字符串"` 默认带有隐藏的结束符 `\0`。在串口传输中，如果不减去这个长度，会把 `\0` 也发出去，导致串口助手显示乱码。
